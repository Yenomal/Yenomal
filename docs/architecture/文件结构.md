# 文件结构

基本结构保持为：
``` text
Yenomal/
├── third_party/
├── datasets/
│   ├── raw/            原始数据
│   ├── product/        经过处理后的数据
│   └── checkpoints/    下载下来的 pre-Train 权重
├── src/
│   ├── common/         经过沉淀可以复用的部分
│   └── projects/       不同 proj/exp 的外部脚本
├── scripts/  
│   └── projects/       不同 proj/exp 的入口 bash 脚本
├── outputs/
│   ├── runs/           不同 proj/exp 的训练后权重
│   ├── eval/           不同 proj/exp 的 eval 结果
│   └── logs/           不同 proj/exp 的日志
├── docs/
│   ├── architecture/   当前 monorepo 的部分信息
│   └── notes/          不同 proj/exp 的过程文件
├── tempwork/           临时工作区，每次结束实验务必清理
├── .gitignore
├── .gitmodule
└── README.md           总信息文件
```
proj/exp 子目录都采用以下方案：
``` text
.../
├── project1/
│   ├── experiment1/
│   └── README.md/
├── project2/
│   ├── experiment1/
│   ├── experiment2/
│   └── README.md/
└── README.md
```
README.md文件最好没有exp,proj下面都配置，记录总结文件，docs里面记录过程文件Research-Plan-List-Run-Process

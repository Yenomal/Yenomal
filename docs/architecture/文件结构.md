# 文件结构

基本结构保持为：
``` text
Yenomal/
├── third_party/
│   ├── AutoGaze/
│   ├── RMBench/
│   ├── SimVLA/
│   ├── hdf5-to-lerobot-converter/
│   └── openpi/
├── datasets/
│   ├── raw/            原始数据
│   ├── product/        经过处理后的数据
│   ├── checkpoints/    下载下来的 pre-train 权重
│   └── lerobot/        转换后的 LeRobot 数据
├── src/
│   ├── pyproject.toml
│   └── yenomal/        经过沉淀并可安装的个人 Python 包
│       ├── nn/
│       └── vision/
├── scripts/
│   ├── data/           数据制作、转换、产品构建入口
│   └── projects/       不同 project / experiment 的入口 bash 脚本
├── outputs/
│   ├── runs/           不同 proj/exp 的训练后权重
│   ├── eval/           不同 proj/exp 的 eval 结果
│   └── logs/           不同 proj/exp 的日志
├── docs/
│   ├── architecture/   当前 monorepo 的部分信息
│   └── notes/          不同 proj/exp 的过程文件
├── tempwork/           临时工作区，每次结束实验务必清理
├── .gitignore
├── .gitmodule
└── README.md           总信息文件
```

各目录职责约定如下：

- `third_party/`
  - 主要工作区，保存各个独立 repo
  - 允许直接改动代码，并通过各自 repo 的 commit / tag 记录实验环境
  - Python 不应主动反推 Yenomal 根目录；跨 repo 条件优先由入口脚本提供

- `src/`
  - 只放可安装、可复用、可被 import 的个人 Python 包
  - 当前统一命名空间为 `yenomal`
  - 安装方式为 `pip install -e ./src`

- `scripts/data/`
  - 保存数据准备相关入口脚本
  - 当前如 `rmbench` 的 meta、norm、LeRobot 转换等都放在这里

- `scripts/projects/`
  - 保存 project / experiment 级别的 train、eval、manifest 入口
  - 入口脚本负责解析 `ROOT`、切换工作目录、拼接 `datasets` / `outputs` 路径

- `datasets/`
  - `raw/` 保留原始数据
  - `product/` 保留可直接复用的数据产品
  - `checkpoints/` 保留外部下载或整理后的初始化权重
  - `lerobot/` 保留转换后的 LeRobot 数据

- `outputs/`
  - `runs/` 保留训练输出与 checkpoint
  - `eval/` 保留评测结果
  - `logs/` 保留运行日志

- `docs/notes/`
  - 记录 project / experiment 的研究、计划、过程与总结

project / experiment 子目录建议采用以下方案：
``` text
.../
├── project1/
│   ├── experiment1/
│   └── README.md
├── project2/
│   ├── experiment1/
│   ├── experiment2/
│   └── README.md
└── README.md
```

补充约定：

- `README.md` 负责记录总结性信息，`docs/notes/` 负责记录过程性信息
- repo 内部资源路径可以在 Python 中通过 `Path(__file__)` 解决
- 跨 repo import 不在 Python 中猜路径，优先由 bash 入口局部提供 `PYTHONPATH`，或直接依赖已安装的 `yenomal` 包
